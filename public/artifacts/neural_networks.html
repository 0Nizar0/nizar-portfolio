Neural Network Architecture

02 / 09
Overview

A neural network is a layered system of interconnected nodes (neurons) that processes input data, transforms it through hidden layers, and produces a predicted output. Each layer learns progressively more abstract features from the data.

Forward Pass

Data flows from input → hidden layers → output. Each neuron applies a weighted sum and activation function to transform the signal.

Backward Pass

The error is calculated at the output and propagated back through the network, adjusting weights to minimize the loss. This is backpropagation.

📐

Layers

03 / 09
Definition

Layers are the structural building blocks of a neural network. Each layer is a collection of neurons that receive inputs, process them, and pass the result to the next layer. The arrangement of layers defines the network's depth and capacity to learn.

🟢 Input Layer

Receives raw data — pixel values, word embeddings, numerical features. No computation happens here; it just passes the data forward. One neuron per input feature.

⚙️ Hidden Layer(s)

Where the actual learning happens. Each hidden layer extracts progressively more abstract features. More layers = deeper network = more complex patterns it can learn.

🎯 Output Layer

Produces the final prediction. The number of neurons here matches the task — one for binary classification, multiple for multi-class, one for regression.


Why Layer Depth Matters

A shallow network (1-2 hidden layers) can approximate simple functions. Deeper networks can represent hierarchical features — like how a CNN's early layers detect edges, middle layers detect shapes, and later layers detect entire objects. Depth is what enables neural networks to solve genuinely complex problems.

🔵

Neurons

04 / 09
Definition

A neuron is the basic computational unit of a neural network. It receives one or more inputs, multiplies each by a weight, adds a bias, and passes the result through an activation function to produce an output signal.

output = activation( Σ(input × weight) + bias )

Inputs

Values received from the previous layer (or raw data if input layer). Each input is multiplied by a corresponding weight before being summed.

Bias

A learnable offset added to the weighted sum. Allows the neuron to shift its activation threshold, giving the model more flexibility to fit the data.

Biological Parallel

Inspired loosely by biological neurons — inputs arrive via dendrites, the cell body processes the signal, and if it exceeds a threshold, an output fires along the axon.

Scale

Modern LLMs can have hundreds of billions of neurons across thousands of layers, all working in parallel to process information simultaneously.

⚖️

Weights

05 / 09
Definition

Weights are learnable numerical parameters attached to every connection between neurons. They determine how much influence each input has on the neuron's output. Training a neural network is fundamentally about finding the right set of weights.

  • Initially set to small random values to break symmetry
  • Updated during backpropagation based on the gradient of the loss
  • A high positive weight amplifies an input's influence
  • A negative weight suppresses or inverts the signal
  • A near-zero weight means the connection has little effect

Weight Initialization

Poor initialization (all zeros or all same value) causes neurons to learn identical features. Random or Xavier initialization ensures diverse learning from the start.

Regularization

Techniques like L1/L2 regularization penalize large weights to prevent the model from over-relying on any single connection — reducing overfitting.

updated weight = weight − learning_rate × gradient
⚡

Activation Functions

06 / 09
Definition

Activation functions introduce non-linearity into the network. Without them, stacking layers would be equivalent to a single linear transformation — no matter how deep the network. Non-linearity is what allows neural networks to model complex, real-world relationships.

ReLU

f(x) = max(0, x)

Most widely used. Fast to compute, avoids vanishing gradients. Outputs zero for negative inputs, passes positive values unchanged.

Hidden Layers

Sigmoid

f(x) = 1 / (1 + e⁻ˣ)

Squashes output to (0,1). Good for binary classification outputs. Can suffer from vanishing gradients in deep networks.

Output (Binary)

Softmax

f(xᵢ) = eˣⁱ / Σeˣʲ

Converts a vector of values into a probability distribution summing to 1. Standard for multi-class classification outputs.

Output (Multi-class)

Tanh

f(x) = (eˣ − e⁻ˣ) / (eˣ + e⁻ˣ)

Outputs between -1 and 1. Zero-centered, which can make training easier than sigmoid in some architectures.

RNNs

Leaky ReLU

f(x) = max(0.01x, x)

Fixes the "dying ReLU" problem by allowing a small gradient for negative inputs instead of outputting zero.

Hidden Layers

Linear

f(x) = x

No transformation. Used in regression output layers where the prediction is a continuous value with no range constraint.

Output (Regression)

Loss Functions

07 / 09
Definition

A loss function measures how far the model's predictions are from the actual labels. It produces a single number that quantifies the model's error on a given batch of data. The entire training process is about minimizing this number.

Mean Squared Error (MSE)

Used for regression tasks. Calculates the average of the squared differences between predicted and actual values. Heavily penalizes large errors.

MSE = (1/n) × Σ(y − ŷ)²

Binary Cross-Entropy

Used for binary classification. Measures the difference between the predicted probability and the actual binary label (0 or 1).

Spam detection Fraud detection

Categorical Cross-Entropy

Used for multi-class classification. Compares the predicted probability distribution to the true one-hot encoded label. Standard for image classification and NLP tasks.

Image classification Language models

Why Loss Function Choice Matters

The wrong loss function sends the wrong signal during training. Using MSE for classification, for example, doesn't properly penalize confident wrong predictions the way cross-entropy does.

🔧

Optimization Algorithms

08 / 09
Definition

Optimization algorithms define how the model's weights are updated during training. They use the gradients computed by backpropagation to move the weights in a direction that reduces the loss. The choice of optimizer and its settings (especially learning rate) significantly impacts training speed and final performance.

Gradient Descent (SGD)

The foundational optimizer. Computes the gradient of the loss across the entire dataset and steps in the negative gradient direction. Slow for large datasets.

Simple tasks

Mini-batch SGD

Computes gradients on small random batches rather than the full dataset. Faster and introduces beneficial noise that can help escape local minima.

Most common

Adam

Adaptive Moment Estimation. Combines momentum and adaptive learning rates per parameter. Generally the best default choice — fast convergence, robust performance.

Industry standard

Learning Rate

The single most important hyperparameter. Too high → training is unstable and overshoots the minimum. Too low → training is painfully slow. Learning rate schedules (warmup, decay) are commonly used in modern training pipelines.

Momentum

Accumulates a velocity vector in the direction of consistent gradients, helping the optimizer accelerate through flat regions and dampen oscillations in ravines. Used by SGD with momentum and Adam.

Summary & Key Takeaways

09 / 09
📐

Layers

The structure. Input, hidden, and output layers stack to form depth. Depth enables complex feature learning.

🔵

Neurons

The units. Each applies a weighted sum + activation to transform and pass signals forward through the network.

⚖️

Weights

The memory. Learnable parameters that encode what the model has learned. Training is entirely about optimizing weights.

⚡

Activation Functions

The non-linearity. Without them, deep networks collapse into a single linear operation. ReLU dominates hidden layers.

📉

Loss Functions

The signal. Measures how wrong the model is. Backpropagation uses this signal to update every weight in the network.

🔧

Optimization

The engine. Gradient descent and Adam translate the loss signal into actual weight updates that improve the model over time.

Key Insight

Every component in a neural network has a specific, irreplaceable role. Remove any one of them and the learning process breaks down. What makes neural networks powerful isn't any single component — it's how they work together across millions of iterations to find patterns in data that no human explicitly programmed.
